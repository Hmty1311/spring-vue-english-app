# S-01 ログイン画面 基本設計書

---

## 1. 画面概要

ユーザー認証を行う画面。認証成功後、モード選択画面（S-02）へ遷移する。

---

## 2. 利用API

| API ID | API名 | 用途 |
| --- | -- | -- |
| AUTH-01 | ログイン認証API | メールアドレス・パスワードを用いて認証を行う |

---

## 3. 表示項目

### 3-1. 表示項目一覧

| 表示名 | 項目名 | 表示 | 備考 |
| --- | -- | -- | -- |
| メールアドレス | email | ○ | メール形式で入力 |
| パスワード | password | ○ | マスク表示 |
| ログインボタン | loginButton | ○ | 押下で認証処理実行 |

---

## 4. 操作・イベント

| 操作 | 内容 |
| -- | -- |
| 画面初期化表示 | 入力欄を初期状態で表示 |
| ログインボタン押下 | 入力値チェック後、AUTH-01を実行 |

---

## 5. バリデーション

| 項目 | 内容 |
| -- | -- |
| メールアドレス | 未入力不可・メール形式チェック |
| パスワード | 未入力不可 |

---

## 6. エラーハンドリング

| ケース | 表示内容 |
| -- | -- |
| 認証失敗 | エラーメッセージを表示 |
| API通信失敗 | 汎用エラーメッセージを表示 |

---

## 7. 画面遷移

| 操作 | 遷移先 | 
| -- | -- |
| ログイン成功 | S-02 モード選択画面 |

---

## 8. 備考

- ログイン成功時にセッション情報を保持する
- 未ログイン状態での他画面アクセスは本画面へリダイレクトする
