# 非機能要件定義書

## 1. 想定利用条件

- 想定利用条件
  1名（個人利用）
- 同時アクセス
  想定しない
- 利用頻度
  日常的な学習利用（1日数回）
- 利用目的
  英単語学習・復習

## 2. パフォーマンス要件

- 画面表示
  通常動作で1秒以内を目標
- API応答
  通常時1秒以内
- クイズ出題
  体感遅延が発生しないこと

## 3. セキュリティ要件

- 認証
  ID＋パスワードによる簡易ログイン
- パスワード
  平文保存は行わず、ハッシュ化して保存
- 認可
  ユーザーは自分のデータのみ操作可能
- 通信
  HTTPS前提（ローカル環境では考慮外）

※JWTやOAuthなどの高度な認証方式は採用しない

## 4. 可用性・信頼性

- 稼働率
  個人利用のため厳密な可用性要件は設けない
- 障害対応
  障害発生時は手動復旧を許容
- データ保全
  定期バックアップは必須要件としない

## 5. 拡張性・保守性

- 構成
  Frontend / Backendを論理的に分離
- 設計
  機能追加・変更が容易な構成とする
- ドキュメント
  doc配下にMarkdown管理

## 6. 動作環境

- クライアント
  PCブラウザ（Google Chrome）
- フロントエンド
  Vue.js
- バックエンド
  Spring Boot
- データベース
  PostgreSQL
- 対応OS
  Windows / macOS（限定しない）
